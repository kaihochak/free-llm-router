# Security Policy

## Reporting a Vulnerability

Please report suspected vulnerabilities privately.

Preferred:

- Use GitHub's private vulnerability reporting for this repository (Security Advisories).

Fallback:

- If private reporting is unavailable, open an issue with minimal details and request a secure contact channel.

Please do not post exploit details publicly until a fix is available.

## What to Include

- A clear description of the issue and impact
- Reproduction steps or a proof of concept
- Affected endpoints, files, or components
- Any suggested mitigations

## Response Expectations

- Initial triage acknowledgement target: within 3 business days
- We will confirm severity, remediation plan, and disclosure timing

## Disclosure

After a fix is released, we may publish a disclosure note and credit the reporter (if requested).
